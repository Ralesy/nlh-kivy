ARCHITECTURE - Архитектура проекта

=== ОБЗОР ===

Проект организован по принципу модульности:
- Каждый модуль отвечает за одну область функциональности
- Чистое разделение ответственности
- Легко расширять и поддерживать
- Нет циклических зависимостей

=== ГРАФ ЗАВИСИМОСТЕЙ ===

=== СТРУКТУРА МОДУЛЕЙ ===

1. UTILS.PY - Вспомогательные функции
   ├─ Константы: RARITIES, RARITY_ICONS
   ├─ Математика: clamp(), rnd_choice_weighted()
   ├─ Утилиты: pause(), clear_screen()
   └─ Вывод: print_header(), print_section()
   
   ⚠️ Зависимостей: 0 (базовый модуль)

2. ITEMS.PY - Система предметов
   ├─ Классы:
   │  ├─ Item (базовый класс)
   │  ├─ Weapon (оружие)
   │  ├─ Armor (броня)
   │  ├─ Potion (зелья)
   │  ├─ Inventory (инвентарь)
   │  └─ ItemDatabase (база данных)
   │
   └─ Методы:
      ├─ Item.display_name() - красивый вывод
      ├─ Item.to_dict() - для сохранений
      ├─ Inventory.add/remove/get() - управление
      └─ ItemDatabase.initialize() - инициализация
   
   ⚠️ Зависимости: utils.py

3. CREATURES.PY - Система существ
   ├─ Классы:
   │  ├─ Creature (враг/нейтралы)
   │  ├─ Player (игровой персонаж)
   │  └─ Companion (спутник)
   │
   ├─ Методы Creature:
   │  ├─ take_damage() - получение урона
   │  ├─ heal() - лечение
   │  ├─ die() - смерть
   │  ├─ equip_weapon/armor() - экипировка
   │  └─ description() - статус
   │
   ├─ Методы Player (дополнительно):
   │  ├─ add_experience() - прокачка
   │  ├─ attack() - атака
   │  ├─ get_session_stats() - статистика
   │  └─ to_dict() - для сохранений
   │
   └─ Поля Player для статистики:
      ├─ total_damage_dealt - выданный урон
      ├─ total_damage_taken - полученный урон
      ├─ enemies_defeated - враги повержены
      └─ battles_fought - битвы проведены
   
   ⚠️ Зависимости: items.py, utils.py

4. BATTLE.PY - Боевая система
   ├─ Классы:
   │  ├─ EnemyGenerator - генератор врагов
   │  ├─ EventSystem - события в локациях
   │  └─ Battlefield - боевое поле
   │
   ├─ EnemyGenerator:
   │  └─ POOLS - враги по локациям:
   │     ├─ forest: 6 врагов
   │     ├─ cave: 4 врага
   │     └─ ruins: 3 врага
   │
   ├─ EventSystem:
   │  ├─ roll_forest() - события в лесу
   │  └─ roll_cave() - события в пещере
   │
   └─ Battlefield:
      ├─ alive_enemies() - живые враги
      ├─ player_attack() - атака игрока
      ├─ enemy_turn() - ход врагов
      └─ attempt_escape() - попытка бегства
   
   ⚠️ Зависимости: creatures.py, items.py, utils.py

5. QUESTS.PY - Система квестов
   ├─ Классы:
   │  ├─ Quest - отдельный квест
   │  └─ Tavern - таверна с квестами
   │
   ├─ Quest:
   │  ├─ register_kill() - убийство врага
   │  ├─ register_win() - победа в бою
   │  ├─ check_complete() - проверка завершения
   │  ├─ claim() - получить награду
   │  └─ progress_display() - отображение прогресса
   │
   ├─ Tavern:
   │  ├─ COMPANIONS - 6 спутников
   │  ├─ available_quests - текущие квесты (5)
   │  ├─ list_quests() - список
   │  └─ get_quest() - получить квест
   │
   └─ Квесты (5):
      ├─ q_kill_wolves - убить 3 волков
      ├─ q_kill_goblins - убить 5 гоблинов
      ├─ q_win3 - выиграть 3 боя (+ меч)
      ├─ q_cave_explorer - 5 побед (+ зелье)
      └─ q_legendary - 10 врагов (+ легендарное оружие)
   
   ⚠️ Зависимости: creatures.py, items.py

6. SHOP_CASINO.PY - Магазин и казино
   ├─ Классы:
   │  ├─ Shop - магазин
   │  └─ Casino - казино
   │
   ├─ Shop:
   │  ├─ list_for_sale() - товары
   │  ├─ buy() - покупка
   │  ├─ sell() - продажа
   │  └─ refresh() - обновить сток
   │
   └─ Casino (статические методы):
      ├─ coinflip() - орёл/решка

8. GAME.PY - Главный модуль игры

   ⚠️ Зависимости: game.py

=== ПОТОК ДАННЫХ ===

Игра:
1. main() → Game.__init__()
2. Пользователь создаёт персонажа → Player()
3. main_menu() → enter_location() → battle_loop()
4. battle_loop() создаёт Battlefield
5. Battlefield использует EnemyGenerator
6. Боевые действия обновляют Player и Creature
7. Квесты отслеживают через Tavern
8. На смерти вывод get_session_stats()

Сохранение:
1. save_game() → to_dict() на Player
2. JSON сохраняется в saves/filename.json
3. load_game() восстанавливает из JSON
4. Все восстанавливается аккуратно

=== ДОБАВЛЕНИЕ НОВОЙ ФУНКЦИИ ===

Пример: Добавить новый тип врага

1. В battle.py → EnemyGenerator.POOLS добавить врага:
   {
       "id": "demon",
       "name": "Демон",
       "hp": 100,
       "dmg": 20,
       "coins": 100
   }

2. В quests.py → создать новый квест (опционально)

3. В game.py → ничего не менять (автоматически работает)

Пример: Добавить новый предмет

1. В items.py → ItemDatabase.initialize() добавить:
   cls.register(
       Weapon("w_demon_sword", "Меч Демона", 5000,
              damage_bonus=50, rarity="Legendary")
   )

2. В quests.py → добавить как награду квеста (опционально)

3. В shop_casino.py → добавить в stock (опционально)

Пример: Добавить новое событие

1. В battle.py → EventSystem.FOREST_EVENTS добавить вероятность:
   ("my_event", 0.05)

2. Добавить обработку:
   if choice == "my_event":
       # логика события
       return "Описание события"

3. В game.py → ничего не менять (автоматически работает)
